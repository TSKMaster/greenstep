# GreenStep

`GreenStep` — конкурсный `MVP` веб-приложения для экологических обращений жителей района.

Проект уже включает рабочее ядро:

- авторизация через `magic link`
- создание новой заявки
- выбор точки на карте
- автоподстановка адреса
- загрузка фото
- просмотр карты обращений
- просмотр списка всех заявок
- базовый профиль пользователя

Остальные части продукта могут оставаться demo-слоем, если это не ломает основной пользовательский сценарий.

## Стек

- `Next.js`
- `TypeScript`
- `Tailwind CSS`
- `Supabase`
- `Leaflet`

## Основные маршруты

- `/` — главная страница
- `/map` — карта обращений
- `/reports` — все заявки
- `/reports/new` — создание заявки
- `/profile` — профиль
- `/auth/sign-in` — вход
- `/preview/main-redesign` — preview-версия для UI-экспериментов

## Что уже реализовано

### Авторизация

- вход через `magic link`
- auth-письма через `Resend`
- поле имени на входе
- сохранение имени пользователя в `profiles.full_name`
- меню пользователя в хедере
- рабочий выход из аккаунта

### Заявки

- создание обращения с категорией, описанием, адресом, координатами и фото
- обратное геокодирование: адрес автоматически подставляется после выбора точки
- обязательные шаги формы перед отправкой
- анонимная отправка

### Карты и списки

- карта обращений с кластеризацией
- цветные маркеры по категориям
- popup-окна заявок
- интерактивная легенда с количеством по категориям
- фильтрация карты по категориям через легенду
- список заявок со статус-фильтрами и счётчиками

### Интерфейс

- новая главная страница уже используется как основная
- единый `header + nav` на ключевых страницах
- мобильная нижняя навигация
- адаптивные CTA, включая `FAB` на мобильных

## Запуск проекта

```bash
npm install
npm run dev
```

По умолчанию приложение будет доступно по адресу:

```txt
http://localhost:3000
```

## Переменные окружения

Нужны как минимум:

```txt
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY
```

## Supabase

В проекте используются основные сущности:

- `profiles`
- `reports`
- `report_comments`
- `report_supports`

Фото заявок хранятся в `Supabase Storage` в bucket `report-photos`.

## Preview

Для безопасных UI-экспериментов оставлен отдельный preview-маршрут:

- `/preview/main-redesign?mode=guest`
- `/preview/main-redesign?mode=auth`

Он нужен для тестирования guest/auth-подачи и новых интерфейсных идей без риска для основного сценария.
