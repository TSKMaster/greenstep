# GreenStep

GreenStep - конкурсный MVP веб-приложения для повышения экологической ответственности жителей района.

## Что делаем

Проект создается как простой и рабочий MVP со следующими принципами:
- только нужный функционал для MVP
- без усложнения архитектуры
- основной упор на работоспособность
- экраны из папки `MVP` реализуются с логикой
- экраны из папки `Demo` реализуются как заглушки

## Технологии

- Next.js
- TypeScript
- Tailwind CSS
- Supabase
- Leaflet

## Текущий статус

Собран рабочий каркас приложения с авторизацией, заявками, картой, статистикой и админкой. Проект уже использует реальные данные из Supabase, развивается как mobile-first web-MVP и использует `Resend` как custom SMTP для auth-писем через `Supabase`.

## Структура проекта

- `src/app` - маршруты приложения
- `src/components` - переиспользуемые компоненты
- `src/lib` - вспомогательная логика, константы, работа с Supabase
- `src/types` - общие типы проекта

## Подключение Supabase

Используются переменные окружения:

```txt
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY
```

В проекте используются клиенты:
- `src/lib/supabase/client.ts`
- `src/lib/supabase/server.ts`

Для auth-писем используется custom SMTP через `Resend`.

## Что уже работает

- вход через magic link
- отправка auth-писем через `Resend` SMTP
- главная страница с реальными данными пользователя и заявок
- создание заявки с сохранением в таблицу `reports`
- клиентская валидация формы создания заявки
- понятные пользовательские сообщения об ошибках для входа, отправки заявки, поддержки заявки и действий в админке
- выбор точки на карте при создании заявки
- загрузка фото к заявке через Supabase Storage
- компактный список всех заявок с переходом в подробную карточку
- компактный список своих заявок с переходом в подробную карточку
- карта заявок на Leaflet
- поддержка чужой заявки через `support_count` с защитой от повторного голоса
- базовая статистика по данным из `reports`
- админка со списком заявок и отдельной страницей редактирования

## Что еще в работе

- demo-разделы: задания, обучение, сообщество, награды
- фиксация схемы базы данных в миграциях
- ручная проверка сценариев на телефоне

## Что важно для фото

В проекте используется public bucket `report-photos`. Код уже готов загружать туда изображения от авторизованных пользователей и сохранять ссылку в `photo_url`.

## Что важно для поддержки заявок

Для защиты от повторной поддержки в базе используется отдельная таблица `report_supports`. Теперь каждая поддержка хранится отдельно, `support_count` обновляется автоматически через триггер, а собственную заявку пользователь поддержать не может.
