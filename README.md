# GreenStep

GreenStep — конкурсный `MVP` web-приложения для повышения экологической ответственности жителей района.

## Что делаем

Проект создается как простой и рабочий `mobile-first MVP` со следующими принципами:

- только нужный функционал для MVP
- без лишнего усложнения архитектуры
- основной упор на работоспособность
- экраны из папки `MVP` реализуются с логикой
- экраны из папки `Demo` реализуются как заглушки
- конкурсная версия может совмещать рабочее ядро и demo-подачу будущих возможностей
- demo-данные допустимы там, где они помогают показать цельный образ продукта для конкурса

## Технологии

- Next.js
- TypeScript

- Leaflet

## Текущий статус

Собран рабочий каркас приложения с авторизацией, заявками, картой, статистикой и админкой.
Проект использует реальные данные из Supabase, развивается как `mobile-first web MVP` и использует `Resend` как custom SMTP для auth-писем через `Supabase`.

Важно: текущая конкурсная версия не стремится сделать все разделы полностью функциональными уже сейчас.
На данном этапе рабочим ядром продукта считаются регистрация / авторизация, создание заявки и базовый просмотр заявок.
Остальные разделы могут быть реализованы как demo-страницы или demo-блоки с понятным описанием того, что планируется внедрять дальше.

Уже подтверждено вручную:

- базовый desktop smoke-check по логике для авторизованного пользователя
- вход через `magic link` на реальном телефоне в локальной сети
- создание заявки с телефона, включая фото
- базовая логика карты, списков и статистики работает

## Структура проекта

- `src/app` — маршруты приложения
- `src/components` — переиспользуемые компоненты
- `src/lib` — вспомогательная логика, константы, работа с Supabase
- `src/types` — общие типы проекта

## Подключение Supabase

Используются переменные окружения:

```txt
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY
```

В проекте используются клиенты:

- `src/lib/supabase/client.ts`
- `src/lib/supabase/server.ts`

Для auth-писем используется custom SMTP через `Resend`.

## MVP и demo

Для этого конкурсного `MVP` важно различать:

- `working core` — регистрация, авторизация, подача заявки, базовый просмотр и карта
- `demo layer` — дополнительные продуктовые разделы, рейтинги, челленджи, расширенное сообщество, обучающие сценарии и другие будущие возможности

Если какая-то часть пока не доведена до полноценной логики, её можно оформлять как demo-сценарий, если это помогает показать развитие продукта и не вводит пользователя в заблуждение относительно реально работающего ядра.

## Что важно для фото

В проекте используется public bucket `report-photos`. Код уже готов загружать туда изображения от авторизованных пользователей и сохранять ссылку в `photo_url`.

## Что важно для поддержки заявок

Для защиты от повторной поддержки в базе используется таблица `report_supports`.
Каждая поддержка хранится отдельно, `support_count` обновляется автоматически через триггер, а собственную заявку пользователь поддержать не может.

## Проверка на телефоне в локальной сети

1. Убедиться, что телефон и компьютер находятся в одной Wi-Fi сети.
2. Открыть приложение по локальному IP компьютера, например:

```txt
http://172.22.0.100:3000
```

3. Для mobile auth flow в `Supabase` должны быть добавлены соответствующие `Site URL` и `Redirect URLs`.
4. Для локального dev-режима в `Next.js` должны быть разрешены нужные dev origins.
