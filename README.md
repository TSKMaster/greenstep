# GreenStep

`GreenStep` — конкурсный MVP веб-приложения для экологических обращений жителей района.

Сейчас проект уже покрывает рабочее ядро:

- авторизация через `magic link`
- создание обращения с категорией, описанием, точкой на карте и фото
- автоподстановка адреса по выбранной точке
- просмотр обращений на карте и в списке
- детальная карточка обращения
- профиль пользователя
- админский сценарий обработки заявок

Остальные разделы могут оставаться demo-слоем, если это честно обозначено в интерфейсе и не ломает основной пользовательский путь.

## Стек

- `Next.js`
- `TypeScript`
- `Tailwind CSS`
- `Supabase`
- `Leaflet`

## Основные маршруты

- `/` — главная страница
- `/map` — карта обращений
- `/reports` — все заявки
- `/reports/new` — создание обращения
- `/reports/[id]` — карточка обращения
- `/reports/success` — экран успешной отправки
- `/profile` — профиль пользователя
- `/admin` — админка
- `/tasks` — demo-страница заданий
- `/learning` — demo-страница обучения
- `/community` — demo-страница сообщества
- `/statistics` — demo-страница статистики
- `/auth/sign-in` — вход
- `/preview/main-redesign` — preview-маршрут для UI-экспериментов

## Что уже реализовано

### Авторизация

- вход через `magic link`
- кастомные письма через `Resend`
- поле имени на входе
- сохранение имени в `profiles.full_name`
- dropdown-меню пользователя в хедере
- рабочий выход из аккаунта
- singleton browser client и middleware для более стабильной сессии

### Обращения

- пошаговое создание обращения
- обязательный выбор точки на карте
- автоподстановка адреса по reverse geocoding
- загрузка фото
- анонимная отправка
- экран успешной отправки

### Карта и список

- отдельная страница карты
- отдельная страница всех заявок
- кластеризация маркеров
- цветные маркеры по категориям
- единые popup-окна
- интерактивная легенда с количеством по категориям
- фильтрация карты по категориям
- фильтры списка по статусам

### Интерфейс

- новая главная уже стала основной
- единый `header + nav` на ключевых страницах
- адаптивная нижняя навигация на мобильных
- обновлённые страницы:
  - главная
  - карта
  - все заявки
  - создание обращения
  - карточка обращения
  - профиль

## Гостевой режим

Для неавторизованного пользователя сейчас доступны:

- главная страница
- публичная карта `/map`
- реальные обращения на карте, если в базе включена policy на публичный `select` из `reports`

Закрытые действия не выполняются напрямую. Вместо этого показывается единая модалка:

`Чтобы использовать данную функцию приложения, необходимо войти.`

Такой сценарий уже работает для:

- создания обращения
- открытия закрытых разделов
- перехода к защищённым действиям из popup и CTA

## База данных

Основные таблицы:

- `profiles`
- `reports`
- `report_comments`
- `report_supports`

Фотографии обращений лежат в `Supabase Storage` в bucket `report-photos`.

Для публичного показа карты гостям добавлена миграция:

- `supabase/migrations/20260413_enable_public_reports_read.sql`

Она открывает только чтение `reports` для `anon` и `authenticated`, не давая гостю права создавать или редактировать заявки.

## Запуск проекта

```bash
npm install
npm run dev
```

Приложение будет доступно по адресу:

```txt
http://localhost:3000
```

```txt
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY
```

## Preview

Preview-маршрут нужен как безопасная песочница для UI-экспериментов:

- `/preview/main-redesign?mode=guest`
- `/preview/main-redesign?mode=auth`

Основную главную без отдельного решения не трогаем через preview-поток. Все спорные редизайн-идеи сначала проверяем именно там.
